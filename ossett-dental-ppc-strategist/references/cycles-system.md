# Recurring Cycles & Goal System

## Overview

This document defines the weekly, monthly, quarterly, and half-yearly optimisation cycles for the Ossett Dental Care AI-First Google Ads Operating System.

---

## 1. Weekly Cycle — Every Monday

**Trigger phrase:** "Run weekly optimisation"

### Tasks to Complete

1. **Review Search Terms**
   - Export search term report
   - Identify irrelevant queries
   - Add new negative keywords
   - Flag potential new exact match opportunities

2. **Analyse Keyword Performance**
   - Review CPC, CTR, CVR by keyword
   - Reduce bids on high CPC / low conversion terms
   - Increase bids on top converters
   - Shift spend toward Hygienist + New Patient clusters

3. **Review RSA Performance**
   - Check headline performance (impressions, CTR)
   - Identify top-performing combinations
   - Pause low-performing asset combinations
   - Queue one new variant for testing

4. **Landing Page Check**
   - Verify page speed
   - Check mobile UX
   - Confirm forms working
   - Confirm phone click-to-call working
   - Check CTA visibility

5. **Tracking Health Check**
   - Verify events firing in GA4 DebugView
   - Check for duplicate events
   - Confirm conversion data flowing to Ads

6. **Extract & Store Insights**
   - Summarise week's performance
   - Note patterns (best hours, devices, keywords)
   - Store in ChatGPT Project memory

7. **Generate Asana Tasks**
   - Create tasks for all recommended actions
   - Assign to Danny
   - Set due dates

### Output Requirements

```
A) Executive Summary (2-3 sentences)
B) Performance Diagnostics (CTR, CPC, CVR, CPL table)
C) Negative Keyword Recommendations
D) Bid Adjustment Recommendations
E) RSA Optimisation Notes
F) CRO Quick-Win Suggestion
G) Tracking Status
H) Asana Tasks (formatted)
```

## 2. Monthly Cycle — Every 10th

**Trigger phrase:** "Run monthly cycle"

### Tasks to Complete

1. **Full Performance Analysis**
   - Month-over-month comparison
   - CPC, CTR, CVR, CPL trends
   - Impression share analysis
   - Device performance breakdown
   - Day/hour performance analysis

2. **Keyword Deep-Dive**
   - Identify expansion opportunities
   - Identify terms to pause/remove
   - Consolidate ad groups if needed
   - Update negative keyword list

3. **Landing Page Review**
   - Review CVR by landing page
   - Evaluate A/B test results (if any)
   - Plan new tests
   - Update copy if needed

4. **Automation Health Check**
   - Verify Make.com scenarios running
   - Check Sheets data accuracy
   - Verify Notion dashboard updating
   - Check ChatGPT memory logs

5. **Generate Monthly Report**
   - Client-facing narrative style
   - Include highlights and lowlights
   - CPL trends with context
   - Keyword insights
   - Recommendations for next month
   - Scaling considerations

6. **Update Notion Dashboard**
   - Push KPIs to client portal
   - Update commentary
   - Archive previous month's data

7. **Update Asana Goals**
   - Update CPL goal progress
   - Update conversion goal progress
   - Update CVR goal progress

### Output Requirements

```
A) Executive Summary
B) Month Performance Table
C) Trend Analysis (vs previous month)
D) Keyword Performance Matrix
E) Landing Page CVR Analysis
F) Automation Status Report
G) Client-Ready Narrative Report
H) Goal Progress Updates
I) Next Month Strategy
J) Asana Tasks
```

**Trigger phrase:** "Run quarterly refresh"

   - Assess search volume shifts

   - Update local intent analysis

   - Update bidding recommendations

4. **Refresh FGACP If Needed**

   - Refresh keyword lists
   - Update RSA templates
   - Review extension performance

5. **Campaign Restructure Evaluation**

   - Evaluate budget split
   - Consider new ad groups

   - Check all events firing
   - Validate conversion imports
   - Identify missing micro-conversions

7. **AI Memory Refresh**

   - Store new quarterly insights
   - Update strategic context

### Output Requirements

```
A) Quarterly Executive Summary
B) Competitive Landscape Update
C) Keyword Universe Refresh
D) Campaign Performance Review (90 days)
E) Recommended Structural Changes
F) Updated MIR Highlights
G) Updated GASP Highlights
```

## 5. Goals Setup

### Goal 1: CPL £31–£37
- Metric: Cost per Lead
- Target: £31–£37
- Red Flag: >£40
- Update: Weekly via automation

### Goal 2: 7–12 Conversions per Month
- Metric: Total Conversions (calls + forms)
- Target: 7–12
- Red Flag: <4
- Update: Weekly via automation

### Goal 3: 8% Landing Page CVR
- Metric: Conversion Rate
- Target: 8%+
- Red Flag: <6%
- Update: Monthly

### Goal 4: Full Automation Deployment
- Metric: Task completion
- Target: All 4 Make scenarios operational
- Update: One-time (project milestone)

### Goal 5: 95% Tracking Accuracy
- Metric: Events firing correctly in GA4
- Target: 95%+
- Validation: Monthly audit
- Update: Monthly

## Performance Alert Thresholds

| Metric | Warning | Critical |
|--------|---------|----------|
| CPL | >£40 | >£50 |
| CVR | <6% | <4% |
| CTR | <3% | <2% |
| Conversions/week | <2 | 0 |
| Call conversions | 0 for 7 days | - |

When thresholds breached:
1. Immediate alert via Make.com
2. Asana urgent task created
3. Root cause analysis required
